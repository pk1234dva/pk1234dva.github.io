SvoManagerMono Class Reference

Detailed Description

MonoBehaviour manager object responsible for scheduling and completing jobs.

It can either load a serialized SvoData object from a scriptable object/binary file, or it can create the SvoData element at runtime.

◆ svoDataBinary

UnityEngine.Object svoDataBinary

Reference to a binary file to determine path from which to load SvoData.

Only used if SvoAcqusitionMethod equals SvoAcquisitionMethod.FromBinaryFile. Has to be contained inside StreamingAssets directory.

Warning
This field is only used for determing the svoDataBinaryName below. In case of deleting or renaming the reference, the string field might not be updated. Make sure to reassign the object again in case you modify the file.

If there are any issues with loading from a binary file, try drag and dropping the the file again into svoDataBinary, and make sure the svoDataBinaryName is then correct.

◆ svoDataBinaryName

◆ collisionLayers

LayerMask collisionLayers

Collision layers used inside FindPath.

Set this to the same value as SvoGeneratorJobs.CollisionCells unless you know what you are doing. The first part of the FindPath method is to find a valid, nearby cell for the input start and end positions. To do this, their neightborhood cells are checked and a a sphere cast is done, until one of those cells is deemed reachable.

◆ ActiveBufferSize

int ActiveBufferSize

Amount of jobs that can be active simultaneously.

◆ CheckValidAgentSize()

bool CheckValidAgentSize ( float  agentRadius)

Function to provide feedback about agent in relation to current svo.

It could happen that an agent uses an SvoManagerMono that is using a very detailed SvoData object - this is problematic, because the found path could potentially go through very small corridors, through which the agent will not fit. This method is meant to be used as a safety check, logging a warning if this is the case.
