Detailed Description

Component for generating an svo object.

Use CreateAndSaveSvo for creating and serializing an Svo in the editor. Use GetSvoData for creating an SvoData at runtime. For more info about the lower level API functions, see the .cs files inside the "SvoGenerateJobsDefinitions" directory.

◆ depth

int depth

Depth of the SvoData object.

Example: depth 0 means that only the base cube would be used. Depth 1 means that if the base cube collides, its 8 subcubes will be used. More depth allows for finding paths in more "narrow" areas. Using an Svo of width 1024 and depth of 8, colliders will be voxelized at depth 1024/(2^8) = 1024 / 256 = 4 - this roughly means that empty corridors of size 4 will be found and be useable. If there is a narrow passage that is only 2 units wide however, it will still not be useable.

More depth exponentially increases the amount of memory needed, and can significantly increase computation time - try to use as low a number as possible that still allows for all the intended paths. In fact, using too much depth could be inappropriate for certain agents - if your agent is a large space ship, it cannot pass through narrow areas - depth should be set, so that the smallest cell size that will be checked for occupancy is still roughly around the size of your agents. This means that if you want to use agents of different sizes, you will have to use multiple generated svo, generated with different depth values.

◆ GetSvoData()

SvoData GetSvoData ( )

Synchronously creates and returns an SvoData with allocated native memory.

Simple wrapper around SvoGenerateJobs.CreateSvoSynchronous that handles the necessary native memory deallocation of auxiliary native structs used inside SvoGenerateJobs.

In case of exceptions, all data will be disposed and the exception reraised.

Warning
Returns an SvoData struct that has allocated native memory and has to be disposed!

Example of use case from SvoManagerMono.Initialize

// GetSvoData cleans up if it fails
case SvoAcquisitionMethod.RuntimeGeneration:
if (svoGenerator == null)
{
SvoLogger.LogError("No svo generator reference set.");
return;
}
svoData = svoGenerator.GetSvoData();
break;
